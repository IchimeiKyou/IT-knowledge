## 🇯🇵 日本語

### プロジェクト概要

**IT就活手册** は、文系・理系問わずITエンジニアを目指す就活生と、日本で働く外国籍エンジニアを対象とした **オールインワンのオフライン対応HTMLガイド** です。

IT技術の基礎から面接頻出質問、ビジネスマナー、日常生活で使う日本語まで、一冊に凝縮しました。専門用語には必ず「わかりやすく言うと」を添え、バーチャルキャラクター **🌸 Momo** と **💻 めいめい** の対話形式でストーリーを展開することで、読者が最初の数段落で「わかる！」と感じて最後まで読み続けられる設計になっています。

### ファイル構成

```
IT-knowledge/
└── index.html   # すべてのコンテンツを含む単一ファイル（外部依存なし）
```

### 収録内容

| パート | 章 | テーマ |
|--------|-----|--------|
| **基礎編** | Ch0 | はじめに / 学習ロードマップ |
| | Ch1 | ハードウェア入門（CPU・メモリ・ストレージ） |
| | Ch2 | IT業界の全体像（SIer・自社開発・スタートアップ） |
| | Ch3 | インターネット基礎（TCP/IP・DNS・HTTP） |
| | Ch4 | プログラミング基礎 / Gitバージョン管理 |
| | Ch5 | データ形式（JSON・XML・CSV） |
| **専門編** | Ch6 | データベース（SQL・インデックス・トランザクション） |
| | Ch7 | クラウド（AWS・IaaS/PaaS/SaaS） |
| | Ch8 | セキュリティ（認証・暗号化・OWASP Top 10） |
| | Ch9 | プロジェクト管理（アジャイル・スクラム） |
| | Ch10 | AI・機械学習（LLM・RAG・ハルシネーション） |
| **実戦編** | Ch11 | 開発プロセス・DevOps（CI/CD・Docker） |
| | Ch12 | インフラ基礎（ロードバランサー・マイクロサービス） |
| | Ch13 | 就活頻出Q&A（自己PR・逆質問） |
| | 付録 | 用語集・システム設計面接入門・英語面接基礎表現 |
| **ビジネス日本語** | Ch17〜22 | 電話応対・客先訪問・会議・ビジネスメール・社内コミュニケーション |
| **生活日本語** | Ch20〜30 | 生活トラブル・交通・銀行・外食・賃貸・職場の人間関係・緊急時対応 |

### 主な特徴

- **ゼロ依存・完全オフライン** — `index.html` をブラウザで開くだけで動作。インターネット接続不要
- **対話ストーリー形式** — 各章冒頭に Momo（文学部就活生）とめいめい（現役SE）の対話を収録
- **実際のニュース事例** — カプコンランサムウェア事件・SQLインジェクション大規模漏洩などの実例で背景を理解
- **SVGインライン図解** — B木インデックス・MLパイプライン・CI/CDフロー等をHTML内に直接埋め込み
- **4段階回答カード** — 面接での「カジュアル→標準→丁寧→謙虚」4レベルの模範解答を横並び比較
- **モバイル対応** — スマートフォンでも読めるレスポンシブレイアウト

### 使い方

1. `index.html` をダウンロード（またはリポジトリをクローン）
2. 任意のブラウザでファイルを開く
3. 左サイドバーから章を選択、または上から順に読む

### 対象読者

- IT業界への就職・転職を目指す文系学生・社会人
- 日本のIT企業で働く外国籍エンジニア
- 技術面接の準備をしたいが専門書は難しいと感じる方

## 🇬🇧 English

### Overview

**IT Job Interview Guide** is an all-in-one, fully offline HTML handbook for **liberal arts graduates and foreign engineers working in Japan**. It covers IT fundamentals, interview Q&A, business etiquette, and everyday Japanese — all packed into a single self-contained HTML file.

Each chapter opens with a story-driven dialogue between two virtual characters — **🌸 Momo** (a literature student) and **💻 Meimei** (a working software engineer) — making technical concepts approachable even for readers with zero technical background.

### File Structure

```
IT-knowledge/
└── index.html   # Single file, no external dependencies, fully offline
```

### Contents

| Part | Chapter | Topics |
|------|---------|--------|
| **Foundations** | Ch0 | Introduction / Learning roadmap |
| | Ch1 | Hardware basics (CPU, memory, storage) |
| | Ch2 | IT industry landscape (SI vendors, in-house dev, startups) |
| | Ch3 | Internet fundamentals (TCP/IP, DNS, HTTP) |
| | Ch4 | Programming basics / Git version control |
| | Ch5 | Data formats (JSON, XML, CSV) |
| **Core Tech** | Ch6 | Databases (SQL, indexes, transactions) |
| | Ch7 | Cloud computing (AWS, IaaS/PaaS/SaaS) |
| | Ch8 | Security (auth, encryption, OWASP Top 10) |
| | Ch9 | Project management (Agile, Scrum) |
| | Ch10 | AI & machine learning (LLMs, RAG, hallucination) |
| **Applied** | Ch11 | Development process & DevOps (CI/CD, Docker) |
| | Ch12 | Infrastructure basics (load balancers, microservices) |
| | Ch13 | Interview Q&A (self-introduction, questions to ask) |
| | Appendix | Glossary, system design interview primer, English interview phrases |
| **Business Japanese** | Ch17–22 | Phone calls, client visits, meetings, business email, internal communication |
| **Daily Life Japanese** | Ch20–30 | Everyday trouble, transit, banking, dining, renting, workplace relations, emergencies |

### Key Features

- **Zero dependencies, fully offline** — Open `index.html` in any browser. No internet or build tools required
- **Story-driven learning** — Each chapter begins with a dialogue using everyday analogies (pizza delivery for Agile, Netflix for cloud elasticity, etc.)
- **Real-world news cases** — The Capcom ransomware incident, SQL injection data leaks, and other real events provide context for why each topic matters
- **Inline SVG diagrams** — B-tree indexes, ML pipelines, CI/CD flows, and more rendered directly in HTML
- **4-level answer cards** — Interview answers shown at four politeness levels (casual → neutral → polite → humble) for side-by-side comparison
- **Mobile-responsive** — Readable on smartphones with a responsive sidebar layout

### Getting Started

1. Download `index.html` (or clone the repository)
2. Open the file in any modern browser (Chrome, Firefox, Safari, Edge)
3. Use the left sidebar to jump to a chapter, or read from top to bottom

### Who Is This For?

- Liberal arts students and career-changers targeting IT roles in Japan
- Foreign engineers working at Japanese IT companies who need business/daily Japanese
- Anyone who finds standard technical textbooks too dense, and wants a story-first approach
